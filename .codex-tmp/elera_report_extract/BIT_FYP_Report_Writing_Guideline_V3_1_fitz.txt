GENERAL GUIDELINES 
 
This document is to be read together with the seminar slides.  
1. A final year project report is an intellectual work for the university. Thus, the written 
report must be in formal English. 
2. Avoid using daily language for the report. For example, do not use the word “I” in the 
report, instead use third person forms (e.g., the developer, the researcher). 
For example: 
Use of Daily Language: (Wrong) 
I have come to conclusion that the system works well with Linux operating system. 
Use of Formal Words: (Correct) 
The researcher concludes that the system works well with Linux operating system. 
3. Any idea, word and reference cited from the Internet, published paper, journal and book 
must be clearly stated by the original author. All work is considered plagiarism if you 
failed to provide any form of reference (In-text citation or reference list) for the report, 
which resulted in an F grade or dismissal from the university. 
 
4. Any term (acronym) used in the report must be clearly defined in full words in the early 
chapter (including glossary page) and then follow with the repeated term in other 
chapters. 
For example: 
The web assessment application (WAA) provides a mechanism to control lab tests and 
quizzes. The WAA has the full capabilities to prevent cheating in formal tests. 
5. Any diagram, figure and table used in the report must be clearly stated/labelled based 
on the chapter and follow with the title. 
For example: 
Chapter 3, first figure the label must be “Figure 3.1 Client-Server Interaction.” Chapter 3, 
second figure the label must be “Figure 3.2 Client Interaction.” 
6. Move any topic/heading or subtopic/subheading at the bottom of the page without any 
content to the new page. 
 
7. The report should have a continuity from one chapter to another; make sure there is a 
consistency of content flow for the report.

3.3 THE NEEDS ANALYSIS 
 
The purpose of conducting the needs analysis to determine whether: 
- 
The learning application is required to fill a gap in public/student/professional knowledge 
and skills. 
- 
E-learning is the best solution to deliver knowledge/training. 
 
The needs analysis allows the identification of general, high-level course goals. 
 
Target audience analysis is a crucial step. The design and delivery of e-learning will be 
influenced by key characteristics of the learners (e.g. their previous knowledge/skills, 
geographical provenance, learning context and access to technology). 
 
Analysis also is needed to determine the course content: 
 
- 
Task analysis identifies the learning tasks that learners/users should follow to improve 
their knowledge and skills that need to be developed. 
- 
Topic analysis is carried out to identify and classify the course content. This is typical of 
those courses that are primarily designed to provide information. 
 
3.4 THE APPLICATION DESIGN 
 
The design stage encompasses the following activities: 
- 
Formulating a set of learning objectives required 
- 
Defining the order in which the objectives should be achieved (sequencing) 
- 
Selecting instructional, media, evaluation and delivery strategies 
 
The outcome of the design stage is a blueprint that will be used as a reference to develop 
the course. The blueprint illustrates the curriculum structure (e.g. college/school courses, 
topics, lessons, activities).
